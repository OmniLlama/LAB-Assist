About Songs, Tracks and Parts

A song is a container for tracks, a track is a container for parts, and a part is a container for events.

A song stores important information like tempo, meter, key, ppq and so on.

A track or a part does not store this kind of information; it is imposed upon them by the song they are added to.

You can create tracks and parts outside the context of a song:

var part = sequencer.createPart();

var track = sequencer.createTrack();

If you add midi events to the part, the events do not have information about bars and beats or minutes and seconds because that information can only be calculated if you know the tempo and the ppq.

var event1 = sequencer.createMidiEvent(0, sequencer.NOTE_ON, 60, 100),
    event2 = sequencer.createMidiEvent(240, sequencer.NOTE_OFF, 60, 0);

part.addEvents(event1, event2);

console.log(part.events[0].bar)             // prints -1
console.log(part.events[0].barsAsString)    // prints N/A
