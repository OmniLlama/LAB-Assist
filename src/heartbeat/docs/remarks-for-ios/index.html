Remarks for iOS

You can use heartbeat on iOS but there are a few things that you have to know.

Unmute WebAudio context

On iOS the WebAudio context is initially muted and needs to be unmuted by user interaction, for instance when the user clicks on a button or taps on the screen. In the eventlistener of such a user interaction you can unmute the WebAudio context, see this article of Paul Bakaus for more details.

In heartbeat there is a special function sequencer.unlockWebAudio(). It is called when you call song.play(), which means that if you are calling song.play() in the eventlistener of a user interaction, the WebAudio context will be unmuted automatically and you don't need to worry about anything.

However if you want to start your song by code, you should call sequencer.unlockWebAudio() on beforehand. This also applies if you call sequencer.processEvent() by code.

Using multiple bitrates

Another issue that you should be aware of is that iOS doesn't seem to like it when you use mp3 files in different bitrates. You should stick to one bitrate for all mp3 files you use in your project. However you can combine mp3 files and wav files. See this example.
